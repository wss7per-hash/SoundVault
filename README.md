# SoundVault

本地 AI 音效库管理工具。把散落在各处的音效文件集中管理，并通过大模型自动生成**使用场景、形象描述、关联关键词和拟声词**，让每个音效都变得「可搜索、可发现」。

> 纯本地桌面软件（Electron），音效文件与数据库都在你自己的机器上，AI 分析需要的 API Key 也仅保存在本地用户目录，不会上传到任何第三方服务器（除你配置的 AI 服务商外）。

---

## 功能特性

- **音效库管理**：导入本地音频目录，自动扫描文件、读取时长等元数据，建立本地 SQLite 索引。
- **AI 语义分析**（核心）：调用大模型把音效「翻译成人话」——
  - 形象描述（日常语言 + 比喻）
  - 使用场景（游戏 / 视频 / UI 等什么时候用）
  - 关联关键词
  - 拟声词（如「金币 → 叮当」「扫射 → 哒哒哒」「水流 → 哗啦啦」）
  - 自动打 3–6 个语义标签，覆盖多个分类维度
- **标签筛选**：左侧标签树，点击即可在右侧过滤出对应音效。
- **音频播放**：内置播放器 + 波形显示，悬停试听；通过私有协议 (`sv://`) 安全加载本地文件，跨 dev / 打包环境一致可用，支持进度拖动。
- **分析可控**：支持单条 / 批量分析，可并发；分析过程中可随时「取消」，互不阻塞。
- **AI 服务商可配置**：内置 OpenAI、DeepSeek、Anthropic、Gemini、Ollama、TokenDance 等预设，也支持完全自定义的 OpenAI 兼容网关。
- **本地 DSP 处理**（无需 AI、零外部依赖）：波形可视化 + 点击试听定位、裁剪截取片段、格式转换（WAV↔MP3）、变速不变调（0.25x–4x 保持音高）、首尾无缝循环。
- **智能组织与检索**：左侧标签树过滤、智能文件夹（按条件自动归集）、收藏夹、批量标签编辑、备注 / 笔记；右侧详情面板展示形象 / 场景 / 关键词 / 拟声词 / 语义标签。
- **库管理**：通过工具栏「库管理 ▾」下拉进行导入、导出（全部 / 自定义范围）、清理重复文件并送回收站；支持全局快捷搜索（Ctrl+Shift+Space）迷你浮层、相似音频推荐。
- **AI 生成音效**：工具栏「AI 生成」按文本提示词调用 Fal.ai / ElevenLabs 生成全新音效并自动入库（按次计费、生成前确认）。
- **声波小精灵（桌面宠物）**：一个常驻桌面的透明音波小精灵，会随音效播放「呼吸」、可点击 / 悬停 / 拖动互动，并与系统托盘统一管理；右键菜单可直达设置 / AI 生成 / 库洞察。


## 安装与运行

要求 **Node.js 18+** 与 **ffmpeg / ffprobe**（用于读取音频元数据，需放在系统 `PATH` 或默认安装位置）。

```bash
# 1. 安装依赖
npm install

# 2. 开发模式（热重载；修改主进程后需手动重启后台 dev 进程）
npm run dev

# 3. 打包（生成 Windows 安装包到 dist/）
npm run dist
```

> 开发环境运行前请确保没有遗留的 `ELECTRON_RUN_AS_NODE` 环境变量，否则 Electron 会退化成普通 Node。

## 配置 AI 分析

首次使用需在设置中填写 AI 服务商信息：

1. 点击工具栏右侧 **⚙ 设置**，在 **模型配置** 中选择服务商（如 TokenDance / DeepSeek / 自定义 OpenAI 兼容网关）。
2. 填入 **API Endpoint** 与 **API Key**（仅保存在本机 `userData`，不入源码、不上传）。
3. 选择模型（如 `deepseek-v3.2`）。
4. 点击「测试连接」确认可用。

> 注意：部分聚合网关的「查模型列表」免费，但「实际对话推理」可能因账户额度不足而返回 402。请先在对应控制台确认有可用额度。

分析时如有网络中断或模型超时，可随时点击「取消」；分析失败不会写入假数据，可重试。

## 数据说明

- 导入音效时，文件会被**复制进本地音效库目录**（默认位于用户数据目录下的 `library/`），原文件不会被移动或删除；数据库保存的是库内副本的路径与元数据索引。
- 本地数据文件（`.db`）、`node_modules/`、`dist/` 等已在 `.gitignore` 中排除，不进入版本库。
- 换机器或重装时注意备份数据库文件（通常位于用户数据目录下的 `soundvault.db`）。

## 声波小精灵（桌面宠物）

SoundVault 内置一个名叫**「声波小精灵」**的桌面宠物：它是一扇常驻桌面的透明小窗，默认停靠在主窗口右下角内侧，会陪你一起听音效、给点小提示。

> 引擎移植自 [duzexu/desktop-pet](https://github.com/duzexu/desktop-pet)（GPL-3.0 兼容），并结合 SoundVault 的音频能力做了定制。

### 它能做什么

- **随声呼吸（音频联动）**：播放音效时，小精灵会跟着音量节奏「呼吸」缩放——声音越大、呼吸幅度越明显；同时进入「演唱」状态：身体随节拍起伏、嘴随响度张合、光晕随峰值脉动。
- **情绪状态**：待机 / 专注（播放时）/ 开心（被戳一下、选中音效时点亮）/ 瞌睡（静置久了）四种情绪，表情、姿势、配色会随之变化（开心是 `^_^` 笑眼 + 暖色调，瞌睡是半眯眼 + 头顶飘小 `z`）。
- **多种互动**：点击、双击、悬停、靠近都会触发不同反应，并弹出气泡对话（点击 / 随机 / 悬停 / 靠近各有一组友情提示与闲聊文案）；静置时还有眨眼、小抖、打哈欠等微动作，更有生命感。
- **丰富的气泡提示**：气泡文案涵盖功能引导（如「试试右键菜单里的 AI 生成音效～」）和日常闲聊，并且以窗口内 HTML 浮层显示，文字清晰不受缩放影响。
- **懂你的库（数据驱动闲聊）**：闲着的时候，小精灵会用你库里的真实统计冒一句友好提示，比如「你库里已经有 N 个音效啦～」「WAV 占了 N 个，是你最常用的格式」「你收藏了 N 个心头好 ♥」。
- **选中点评**：在主窗口选中任意音效，小精灵会用它的 AI 分析结果（形象描述 / 使用场景 / 拟声词）念一句点评，并乐一下。
- **拖卡片试听（B2）**：把主窗口里的音效卡片拖到小精灵身上松手，主窗口就会试听那个音效，小精灵同步跟着演唱——随手挑音效超方便（拖拽到宠物窗口上方时，小精灵会亮起一圈虚线提示「松手就能试听啦～」）。
- **可暂停互动**：右键菜单可随时「暂停互动 / 恢复互动」，安静工作时让它乖乖待着。
- **自由拖动**：用鼠标按住即可拖动，支持跨屏幕移动；松手后位置自动保存，下次启动仍在原位。
- **系统托盘统一管理**：小精灵与主程序共用一个**品牌托盘图标**（左键 / 右键 / 双击均可弹出菜单），不再有独立托盘。托盘菜单以主窗口控制为主、宠物为子区。
- **右键快捷入口**：在小精灵上右键，原生 OS 菜单提供——打开设置、隐藏 / 显示、切换置顶、AI 生成音效、库洞察、重新引导、退出。
- **换肤（A3）**：在「设置 → 宠物（声波小精灵）」面板里，除了色相 / 缩放 / 透明度滑块，还提供一组预设皮肤胶囊（经典紫 / 薄荷绿 / 天空蓝 / 蜜桃粉 / 暖阳橙 / 葡萄紫），一键切换小精灵配色。
- **新手引导（B3）**：首次启动会自动弹出分步气泡，教你拖动、右键菜单能做什么、播放时它会跟着唱歌；随时可在右键菜单点「重新引导」再看一遍。

### 配置项（本地持久化）

宠物偏好存于本地设置（`settings` 表 `pet.config`），包括：

| 配置 | 说明 | 默认 |
| --- | --- | --- |
| `scale` | 精灵整体缩放 | `0.35` |
| `visible` | 是否显示宠物 | 开 |
| `alwaysOnTop` | 是否置顶 | 关 |
| `audioBreath` | 是否随声呼吸 | 开 |
| `paused` | 是否暂停互动 | 关 |
| `x` / `y` | 窗口位置 | 主窗口右下角内侧 |

所有配置仅保存在本机，不会上传。

### 开关与入口

- 工具栏「🐾 小精灵」按钮可显示 / 隐藏宠物。
- 系统托盘菜单同样可以控制显示、置顶、退出。
- 宠物窗口关闭不会退出主程序；主程序退出时宠物随之关闭。

## 未来开发计划（Roadmap）

以下为迭代中的功能规划，已实现的会明确标注 ✅：

### 1. 提示词生成音效（text-to-SFX）✅ 已实现
- 根据提示词调用文本→音效生成模型，产出全新音效并自动入库。
- 已实现 **Fal.ai（Stable Audio 2.5，最长 190s）** + **ElevenLabs SFX V2（最长 20s）** 双服务商，按次计费、生成前确认花费、免费试用额度；配置独立于 AI 分析，位于设置面板「AI 生成」。
- 入口：工具栏右侧 **AI 生成**（魔杖图标）。

### 2. 首尾无缝循环 ✅ 已实现
- 通过 ffmpeg 交叉淡化（crossfade）/ 自动寻找最佳衔接点，实现本地音频无缝循环，已并入音效库。

### 3. 生成类似音频（audio-to-audio / 参考生成）
- 参考输入音效，生成风格 / 内容相近的新音效。
- 可行路线：将本库 AI 分析结果（使用场景 + 拟声词 + 标签）作为 prompt 喂给 text-to-SFX 模型，近似实现「类似」。
- 更彻底的 audio-to-audio 参考生成目前模型尚不成熟，暂缓（留待 V2）。

### 4. 延长当前音频（extension）✅ 低成本方案已实现
- 真·AI 续写：Stable Audio inpainting（质量待评估），暂缓。
- 低成本方案：**变速不变调（DSP 时间拉伸，0.25x–4x 保持音高）** 已落地，多数「延长 / 放慢」需求可由此满足，无需生成模型。

> 另有已交付的本地 DSP 与检索能力（均在功能特性与手册中详细介绍）：波形可视化 + 点击试听定位、裁剪截取片段、格式转换 WAV↔MP3、批量标签编辑、备注 / 笔记、全局快捷搜索（Ctrl+Shift+Space）、相似音频推荐、库内重复文件查找与清除、库洞察统计面板与标签云。

### 备注
- 生成类功能（1 / 3 / 4）需接入外部音频生成 API 或本地部署生成模型（Stable Audio / AudioCraft 开源权重），按次计费或需 GPU 算力。
- 音乐生成模型（Suno / Udio）偏音乐且 API 受限，与游戏 / UI 音效库场景不匹配，暂不纳入。
